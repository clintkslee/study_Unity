오브젝트 + 컴포넌트

컴포넌트
게임 오브젝트에 부착할 수 있는 C# 스크립트 파일을 지칭하는 단위
게임 오브젝트에 컴포넌트를 부착되어 게임 오브젝트에 여러 기능을 부여

<주요 컴포넌트>

트랜스폼
이동, 회전, 크기조절

렌더러(sprite=2d, mesh=3d) 
변수에 이미지 에셋 등록하여 화면에 오브젝트 출력

오디오 소스
오디오클립 변수에 등록된 사운드 에셋 재생

콜라이더
오브젝트의 충돌 범위

C#스크립트파일
개발자가 지정한 고유 행동(이동, 공격, 피 없을 시 도망 등)

컴포넌트는 에셋스토어에서 구매도 가능


***

에셋 = 오디오, 3d모델, 애니메이션, 텍스쳐, 스크립트, 프리팹 등

프리팹 : 게임 오브젝트를 파일 형태로 저장하는 단위
주로 게임 중간에 생성되는 게임 오브젝트를 프리팹으로 저장해두고 사용
장점 : 동일 게임오브젝트 여러 씬, 장소에 배치 시 쉽게 복붙 가능
원본 데이터 수정 시 배치된 모든 게임오브젝트에 반영

***

유니티의 구조
1 프로젝트 = n 씬
1 씬 = n 게임오브젝트
1 게임오브젝트 = n 컴포넌트+에셋

***

유니티2d 좌표 체계
위쪽(y), 오른쪽(x)

유니티3d 좌표 체계
왼손좌표계 - 정면(z), 위쪽(y), 오른쪽(x)


***

카메라
씬에 최소 1개 이상 카메라 오브젝트 배치
- 카메라 컴포넌트 : 눈 역할
- 오디오 리스너 컴포넌트 : 다른 게임오브젝트의 오디오 소스 컴포넌트가 내는 소리 듣는 역할

- 클리어 플래그 : 오브젝트 없는 빈 배경을 무엇으로 채울 지 결정
2d solid color -> background 조절로 색상 변경
3d skybox
클리어플래그가 depth only, don't clear로 설정 시 오브젝트 이동하면 잔상이 남는다

- 프로젝션
카메라의 시점, 2d와 3d 시점 존재
2d : z축은 오브젝트의 전후 관계만 영향
3d : 멀리 있는 오브젝트는 작게 보임(원근투영)

- 클리핑 플래인스
카메라가 볼  수 있는 시야 거리
near : 시작지점(0.3), far : 끝지점(1000)
far 거리 상향 시 카메라가 볼 수 있는 시야 거리 증가(= 화면에 표시되는 오브젝트 증가 = 성능에 영향)

- 뷰포트 렉트
카메라가 본 것을 화면에 출력하는 영역 설정
x, y : 0~1의 범위, 출력을 시작하는 위치 좌표
width, height : 0~1의 범위, 출력하는 화면 크기
(x=0,y=0), (w=1,h=1) 설정 시 한 카메라로 모든 화면 사용
-> 카메라 여러 대 배치 시 뷰포트 조절하여 한 화면에 여러 화면 띄울 수 있다.

라이트 컴포넌트
현실세계의 빛 역할, 유니티는 4가지 종류 빛 제공
- 환경광, 주변광(윈도우-렌더링-라이팅 세팅),카메라의 클리어 플래그까지 꺼줘야 완전한 암전 설정 가능

빛의 종류
디렉셔날 라이트 : 모든 오브젝트에게 동일한 방향으로 빛을 제공
포인트 라이트 : 구형태 방사형 빛
스폿 라이트 : 원뿔 형태 뻗어나가는 빛
에어리어 라이트 : 오브젝트 기준 전방으로만 방출(baked mode - 미리 빛의 데이터 지정해놓고 사용하는 방식으로만 사용 가능)

빛의 모드
리얼타임 : 게임 중 빛의 이동, 변화 실시간 연산
믹스드 : 리얼타임과 베이크드 섞어 사용
베이크드 : 빛의 이동, 변화x

인텐시티 : 빛의 세기 

쉐도우 타입 : 빛을 받은 오브젝트의 그림자 설정

쿠키 : 빛의 모양을 등록된 이미지의 모양으로 생성
