Tilemap 
타일 형태의 리소스를 배치해 게임 월드를 표현하는 방법

***
유니티는 타일맵 에디터를 제공

Tile Palette = 팔레트
타일맵 오브젝트에 배치할 타일 에셋들들을 등록해두는 저장소
팔레트에 있는 타일 에셋을 여러 속성에 따라 배치하거나 삭제 가능

Tile Asset = 물감
맵에 배치할 수 있는 하나의 타일(스프라이트, 컬러, 콜라이더 타입 같은 속성 설정 가능)

Tilemap 오브젝트 = 도화지
타일 에셋을 배치하는 공간, 실제 게임에 보여지는 타일 형태의 게임 월드
그리드 오브젝트의 자식으로 여러 개의 타일맵 오브젝트 등록 가능 (Layer 구분)

Grid 오브젝트
자식으로 배치되는 타일맵 오브젝트들을 관리하는 역할
cell layout, cell swizzle 정보를 이용해 배치되는 맵의 방식을 rectangle, hexagon, isometric, isometric z as y 등으로 설정 가능

*** 타일 팔레트 생성
윈도우-2D-타일 팔레트에서 새 팔레트 생성
name : 팔레트 이름
grid : 타일 배치 방식
cell size : 셀의 크기
저장 위치 Asset 폴더 내여야 한다

*** 타일 에셋 생성
프로젝트뷰 + 2D 타일 로 생성
sprite : 현재 타일 에셋이 보여지는 이미지
color : 이미지의 색상
collider type : 충돌 속성(none, sprite, grid)

*** 타일에셋을 타일 팔레트에 등록
팔레트에 에셋을 드래그

*** 타일 배치
계층뷰에서 타일맵 오브젝트 생성 -> 그리드오브젝트의 자식으로 생성됨

Grid 게임오브젝트 : 자식으로 있는 타일맵 게임오브젝트들을 관리
타일맵 오브젝트는 게임에서 충돌 가능하며, 실제 맵에 배치되는 플랫폼 타일, 배경으로 사용되는 타일, 아이템 타일 등 여럿으로 나누어 제작

Grid 컴포넌트 : 
Cell Size : 그리드 내 각 셀의 크기
Cell Gap : Grid 내의 각 셀 사이의 간격
Cell Layout : 셀의 나열 방식
Cell Swizzle : 축의 나열 변경 - Swizzle이 xyz일 때 (0,1,1)dlaus, Swizzle을 yxz로 바꾸면 (1,0,1)이 된다

* 타일맵 오브젝트의 컴포넌트들
Tilemap 컴포넌트 : 
현재 타일맵 오브젝트에 대한 기본 설정 (타일의 색, 각 타일의 중심점, 방향 등)
Scene view에서 현재 타일맵 오브젝트에 표현될 타일들을 그릴 수 있다

Tilemap Renderer 컴포넌트
현제 타일맵 오브젝트가 화면에 그려지는 방법에 대해 설정(material, 정렬 기준, 순서 등) `12`				

타일맵의 생김세는 씬뷰에서 확인 가능 (그리드 오브젝트의 그리드 컴포넌트가 형태 결정)		
형태 : 사각형, 육각형(가로, 세로), 마름모		

*** 타일 그리기
타일 팔레트에서,
타일 선택 : S
타일 이통 : M (타일팔레트, 타일맵 오브젝트(씬뷰)에서)
* 타일 팔레트 내부의 타일 에셋 설정, 삭제 시 Edit(오른쪽 상단 첫번째) 누르고 하기
B : 타일 배치(Edit 기능 풀고), 계층뷰에서 원하는 타일맵오브젝트 선택 후 배치
U : 타일 배치2(사각형 범위 만큼 배치, Shift 누르고 사용시 삭제)
I : 타일 팔레트나 타일맵 오브젝트에 배치된 타일 복제하여 선택
D : 타일 팔레트나 타일맵 오브젝트에 있는 타일 삭제
G : 타일 팔레트에서 선택한 타일을 타일맵 오브젝트에 배치(그림판의 물감통)

***
타일맵으로 그려진 게임월드는 타일맵 오브젝트 단위로 충돌처리 가능
(그리드 오브젝트는 여러개의 타일맵 오브젝트를 관리)

타일맵 오브젝트에 Tilemap Collider 2D 컴포넌트 추가
- 프로젝트뷰의 타일 에셋의 기본 콜라이더가 sprite면 그림 모양따라 콜라이더 생김, grid로 설정 시 그리드 모양대로 콜라이더 생김

더 자세한 콜라이더 처리 위해 Composite collider 2D 추가
(타일 콜라이더 2D 컴포넌트에 used by composite 체크, Rigidbody 2D의 바디타입 스태틱으로 설정)
-> 붙어있는 타일들이 하나의 콜라이더로 설정된다.
